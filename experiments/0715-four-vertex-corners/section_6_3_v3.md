# 6.3 v3：四顶点极坐标入口下的拓扑感知环形分区

为什么要做 v3：前面的 v2 已经把问题从连续变形转为“层归属、层内顺序、角宽边界”的离散拓扑优化，但它的几何入口仍以单元中心为主。0715-角点实验把 Step 1 改为四顶点极坐标：先从原始经纬度中心和网格步长重建每个 cell 的四个角点，再分别计算每个角点的 theta、rho 和环形半径 R，形成真实的四顶点极坐标多边形。随后不直接把这个多边形当作最终区域，而是提取四顶点多边形的质心、方向和半径，继续进入 Step 2-4 的拓扑感知流程。

这个改动的意义在于：v3 让初始几何不再只是“中心点 + 固定小方格”的诊断符号，而是先保留每个网格单元在原始经纬度下的角点尺度与边界倾向。这样可以更早暴露边界单元、狭长区域和方向密集区带来的几何问题，同时又不把四顶点投影的越界、面积畸变和局部挤压直接固化为最终视觉区域。换句话说，四顶点极坐标提供更诚实的几何入口，最终分区仍由拓扑、容量和角宽约束重新生成。

实现上，v3 对 workflow 做了三处连锁调整。第一，Step 1 额外输出 four_vertex_grid_cells.csv、four_vertex_grid_corners.csv 和 step01_four_vertex_metrics.csv，用于记录四角点、多边形质心和基础几何诊断。第二，Step 2 优先读取四顶点生成的 cell 表，并在拓扑图中使用真实四顶点多边形辅助检查，而不是只显示中心符号。第三，Step 3 与 Step 4 的输入统一切换为 four_vertex_grid_cells.csv，使后续分层、邻接分类、层容量分配、排序优化和角宽优化都基于同一套 v3 几何入口。

当前运行结果显示，四顶点入口下的初始有效邻接 F1 为 0.576，优化后提升到 0.711。优化后在 312 条原始四邻接边中保留 254 条，丢失 58 条，新增 148 条；有效邻接 precision / recall / F1 分别为 0.632 / 0.814 / 0.711。跨多层冲突从 32 条降到 11 条，层容量误差从 0.213 降到 0.00619。层容量由初始 [30,29,29,29,29,30] 调整为 [23,25,28,32,34,34]，接近按环带面积得到的目标 [22,25,28,31,34,36]。

| 指标 | v3 初始 | v3 优化后 | 解释 |
| --- | ---: | ---: | --- |
| 保留原始边 | 217 | 254 | 更多原始四邻接获得有效接触 |
| 丢失原始边 | 95 | 58 | 丢失比例从 30.45% 降至 18.59% |
| 新增邻接 | 225 | 148 | 非原始接触明显减少，但仍存在 |
| 邻接精确率 | 0.491 | 0.632 | 最终邻接中原始边占比提高 |
| 邻接召回率 | 0.696 | 0.814 | 原始边被保留的比例提高 |
| F1 | 0.576 | 0.711 | 拓扑保留的综合指标提升 |
| 跨多层冲突 | 32 | 11 | 层移动显著缓解长距离冲突 |
| 层容量误差 | 0.213 | 0.00619 | 分层更接近环带面积容量 |

代价也需要保留在结论中。v3 优化后平均角度偏移为 7.58°，中位数 7.20°，最大 28.40°；56 个单元偏移超过 10°，17 个超过 15°。最小角宽被压到 1.5°，有 11 个单元触及下界；8 个单元超过最大目标宽度；43 个单元的形状比超出设定范围。因此，v3 的结果应解释为“在四顶点入口下，拓扑保留和容量匹配进一步改善”，而不是“几何形状已经完全均衡”。

与 v2 的关系应这样理解：v3 没有推翻 v2 的三层优化逻辑，而是把 v2 的输入几何从中心点参考升级为四顶点参考。它回答的是另一个更细的问题：如果入口阶段更接近真实网格单元边界，后续拓扑感知分区是否仍然稳定、是否能得到更好的邻接结果。当前答案是肯定的，但仍需要下一步继续处理窄角宽、形状比异常和角度偏移的约束平衡。
